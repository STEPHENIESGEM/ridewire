# Contributing to RideWire

Thank you for your interest in contributing to RideWire! We welcome contributions from everyone and appreciate your effort to improve this project.

## How to Contribute

We appreciate all contributions, whether you're fixing bugs, adding features, or improving documentation. Please follow these steps:

1. **Fork the repository** to your GitHub account
2. **Create a feature branch** (`git checkout -b feature/amazing-feature`)
3. **Commit your changes** with clear, descriptive messages (`git commit -m 'Add amazing feature'`)
4. **Push to your branch** (`git push origin feature/amazing-feature`)
5. **Open a Pull Request** with a detailed description of your changes

## Code of Conduct

We are committed to fostering an inclusive and welcoming community. All contributors are expected to:

- Be respectful and inclusive of different viewpoints and experiences
- Provide constructive feedback and be open to receiving it
- Ask questions when unclear rather than making assumptions
- Help and support other community members
- Focus on what is best for the community and the project

## Reporting Issues

Use GitHub Issues to report bugs or request features. To help us address your issue quickly, please include:

- **Clear description**: Explain the issue concisely
- **Steps to reproduce**: For bugs, provide detailed reproduction steps
- **Expected vs. actual behavior**: What should happen vs. what actually happens
- **Environment details**: Your OS, Python version, and other relevant information
- **Screenshots or logs**: If applicable, add visual aids or error logs

## Pull Request Process

Before submitting a pull request, please ensure:

1. **Code quality**: Your code follows the project's style guidelines and best practices
2. **Documentation**: Update documentation to reflect any new features or changes
3. **Testing**: Add or update tests to cover your changes
4. **Issue linking**: Reference any related issues in your pull request description
5. **Review readiness**: Request review from maintainers once your PR is ready

We will review your pull request as soon as possible and provide constructive feedback.

## Questions?

If you have questions about contributing, feel free to:

- Open a GitHub issue with the "question" label
- Start a discussion in [GitHub Discussions](https://github.com/STEPHENIESGEM/ridewire/discussions)
- Contact us at support@ridewireai.com

We'll get back to you as soon as possible. Thanks for contributing to RideWire!
